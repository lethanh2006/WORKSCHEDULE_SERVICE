## Goal Description
Implement the complete backend API and database models for the `workschedule` microservice based on the specifications. This includes Work Policy management, Schedule Requests (User & Admin views), and Attendance via QR checking.

## User Review Required
> [!IMPORTANT]
> **Primary Key Standard:** The specification states `PK = uuid`. Typically in MongoDB (used by your other apps), the standard primary key is `ObjectId`. I plan to use standard `mongoose.Types.ObjectId` to maintain consistency across the system, but I can use string UUIDs generated by `crypto.randomUUID()` if expressly required. 
> Please confirm if using default Mongoose `ObjectId` is acceptable, or if you strictly want UUID `string` types.
> 
> **User Service Integration:** You mentioned needing to call the `user` service. I will build an HTTP client using `axios` in `workschedule` to make requests to the user service securely (likely verifying the internal JWT/secret or passing the request token over). Please let me know what port/URL the `user` service is accessible at if differ from `localhost:5001`.
